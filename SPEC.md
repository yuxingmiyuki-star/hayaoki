# メザミー（Mezamy）— 設計図

> **早起き版デュオリンゴ。** 毎朝つづける「早起き学習アプリ」。
> 相棒キャラ・連続記録🔥・デイリークエスト・XP・早起きの道で習慣化し、起床カードをTwitter(X)に自慢できる。
> `jikan-app` リポジトリの流儀（単一 `index.html` / React (CDN) / localStorage / ビルド不要 / PWA）に準拠。

---

## 0. この文書について

実装前の **設計図（仕様書）**。コンセプトは段階的に進化してきた：
「健康改善アプリ」→「早起きをゲーム化してTwitter共有」→ **「早起き版デュオリンゴ（毎日つづく習慣化アプリ）」**。

主役はデュオリンゴ式の**習慣化エンジン**（ストリーク／デイリークエスト／マスコット／パス／XP）。
そこに**Twitter共有**を組み合わせ、サーバ無しでも"ゆる競争"が成立するよう設計する。

---

## 1. デュオリンゴ式：何を真似て、何は割り切るか

デュオリンゴの中毒性を分解し、単一HTML（ローカル完結）でどこまで再現できるかを整理した。

| デュオリンゴの要素 | 本アプリでの姿 | ローカルだけで実現 |
|---|---|---|
| 連続記録🔥＋ストリークフリーズ | 連続早起き＋「寝坊チケット」 | ✅ 可（習慣化の核） |
| XP・レベル・デイリーゴール | 起床ポイント・レベル・今日の目標 | ✅ 可 |
| マスコット（褒める/煽る） | 相棒キャラ（ニワトリ）が反応 | ✅ 可 |
| 学習パス（道なりに進む） | 「早起きの道」進行マップ | ✅ 可 |
| デイリークエスト | 毎朝のミッション | ✅ 可 |
| 達成演出・バッジ | そのまま | ✅ 可 |
| しつこいリマインド通知 | 相棒の声で通知 | △ アプリ非起動時のWeb通知は弱い |
| **リーグ／ランキング（他人と競う）** | 週間リーグ | ❌ 要バックエンド |
| フレンド・複数端末同期 | — | ❌ 要バックエンド |

**結論**：デュオリンゴ体験の大半（ストリーク・XP・マスコット・パス・クエスト）はローカルで再現可能。
唯一「リーグ」だけがサーバ案件。**MVPはTwitter共有による"ゆる競争"で代替し、本物のリーグはフェーズ4で後付け**する（→9・11章）。

## 2. コンセプト

**「起きたら、勝ち。相棒と毎日つづける。」**

つらい早起きを、相棒キャラと一緒に積み上げる"毎日の習慣"に変える。
起きた瞬間にド派手に祝い、連続記録を伸ばし、起床カードをTwitterに自慢して仲間とゆるく競う。

### 楽しさの3本柱
1. **毎朝の達成ループ**：起きる→クエスト達成→XP・連続記録が伸びる→相棒が喜ぶ。
2. **積み上げと進行**：早起きの道を進み、レベル・称号・バッジ・箱庭が育つ。
3. **自慢と共感**：起床カードをTwitterへ。ハッシュタグで朝活仲間とつながる。

---

## 3. マスコット（相棒キャラ）★デュオリンゴの肝

デュオリンゴのフクロウに当たる存在。**朝テーマなのでニワトリ**を提案（仮名「コケ太」）。

- **役割**：毎朝あなたを起こし、起床を褒め、サボると悲しむ"感情のフック"。
- **性格**：明るい体育会系。早起き成功で全力で喜ぶ／二度寝・寝坊で「コケ…」としょんぼり／連続が途切れそうな日は煽り＆応援。
- **表情ステート**：`happy / cheer / sleepy / sad / panic（連続途切れ警告）` を顔差分で表現（絵文字＋CSS、または軽量SVG）。
- **セリフ**：状況に応じた固定文言テーブルからランダム。共有カードにも顔とひとことが載る。
- **着せ替え（将来）**：レベルやバッジで帽子・背景をアンロック。

> ねらい：数値だけでなく「相棒をがっかりさせたくない」という罪悪感ドリブンで継続を促す（デュオリンゴの最大の発明）。

## 4. 早起きの道（学習パス）＆デイリークエスト

### 4.1 早起きの道（進行マップ）
- デュオリンゴのレッスンパスに相当。**1日＝1ステップ**で道を進む縦スクロールのすごろく風マップ。
- 数ステップごとに「チェックポイント」（バッジ獲得・箱庭が発展・相棒の新セリフ解放）。
- 連続成功で道が分岐・装飾が豪華に。見た目で"積み上げ"を実感させる。

### 4.2 デイリークエスト（毎朝のレッスン）
その日の「やること」。達成でXP。例：
- **必須**：目標時刻までに「起きた！」を押す（コアクエスト）。
- **任意ミッション**：カーテンを開けて光を浴びる☀️／コップ1杯の水💧／軽くストレッチ🤸／顔を洗う。
- 1つでも多く達成すると**コンボボーナス**。全部やると「パーフェクトモーニング」で相棒が大喜び＋特別カード。

## 5. ゲームデザイン（中核ロジックの考え方）

### 5.1 起床フロー（毎朝の体験）
```
通知で起こす（相棒の声）
  ▼ アプリを開く
巨大な「起きた！」ボタンをドーン
  ▼
演出（紙吹雪🎉＋効果音＋相棒のリアクション）
  ▼
デイリークエストのチェック（任意ミッション）
  ▼
スコア計算 → XP/レベル/連続記録更新・バッジ判定・道が1歩進む
  ▼
起床カード生成 →[Twitterに自慢]/[画像保存]/[閉じる]
```

### 5.2 スコア／XP（実装の目安）
```
base = 100
early        = max(0, goalMinutes - wokeMinutes)   // 早く起きた分
streakBonus  = min(streak, 30) * 10
missionBonus = missions.length * 20
snoozePenalty= snooze * 15
score = max(0, base + early + streakBonus + missionBonus - snoozePenalty)
XP += score
```

### 5.3 連続記録🔥＋ストリークフリーズ
- 目標時刻（＋猶予）までに起床で成功 → streak +1。
- 失敗日は streak リセット。ただし**寝坊チケット（freeTickets）**が1枚あれば自動で連続を1日維持。
- チケットは連続◯日ごと、または特定バッジで補充。

### 5.4 レベル・称号・バッジ
- `level = floor(sqrt(XP / 100))` 程度の緩い曲線。レベル帯ごとに称号（「朝5時の住人」「日の出マスター」等）。
- バッジ：「3日坊主突破」「皆勤賞(7日)」「30日継続」「6時前クラブ」「パーフェクト連発」など。獲得で演出＋共有導線。

### 5.5 箱庭（任意のやり込み）
- 早起きするほど「朝の街」が発展。純ローカル演出でサーバ不要。

## 6. Twitter(X) 共有設計（リーグの代役）

サーバ無しでも"社会的プレッシャー"を生む仕掛け。

### 6.1 起床カード（シェア画像）
- **Canvas** で動的生成。要素：起床時刻 / 連続記録🔥 / レベル・称号 / 今日のスコア / 相棒の顔＋ひとこと / 日付 / ハッシュタグ。
- 時間帯で背景が朝焼け⇄昼に変化。1080×1080 と 1200×675 を用意。

### 6.2 共有の3段フォールバック
1. **スマホ**：`navigator.share({files,text})`（Web Share API）で画像＋テキストを直接共有。
2. **PC/非対応**：**Twitter Web Intent**（`intent/tweet?text=...&hashtags=...`）＋画像ダウンロード。
3. それも不可なら**クリップボードにコピー**＋画像保存。

### 6.3 共有テキスト例
```
おはよう！今日は6:12に起きた☀️ コケ太も大喜び🐔
🔥7日連続  Lv.5「朝5時の住人」  今日のスコア 320pt
#メザミー #早起きチャレンジ
```

### 6.4 ゆる競争
- 共有文に連続日数・スコアを必ず入れる → ハッシュタグが事実上のランキング場に。
- アプリ内「#メザミーを見る」でハッシュタグ検索を開き、仲間の投稿を覗ける。

```jsonc
{
  "version": 1,
  "profile": {
    "name": "ゆき",
    "goalWakeTime": "06:30",
    "mascot": "koketa",          // 相棒（将来は着せ替え状態も）
    "createdAt": "2026-06-26",
    "settings": { "sound": true, "notify": true, "shareSuffix": "#メザミー #早起きチャレンジ" }
  },
  "game": {
    "xp": 1280, "level": 5, "title": "朝5時の住人",
    "streak": 7, "bestStreak": 12, "freeTickets": 1,
    "pathStep": 23,              // 早起きの道の現在地
    "badges": ["start3","week7","before6"],
    "village": { "stage": 3 }
  },
  "wakeups": [
    {
      "date": "2026-06-26", "wokeAt": "06:12", "goalAt": "06:30",
      "snooze": 0, "missions": ["light","water"], "perfect": false,
      "score": 320, "shared": true
```

## 10. 技術設計

`jikan-app` の既存パターンに準拠。

- **単一 `index.html`**：HTML+CSS+JS同梱。インストール/サーバ不要。
- **React (CDN)** + `babel-standalone`（ビルド不要）。
- **状態管理**：`useReducer` + `useLocalStorage`（`mezamy:v1` 永続化／マイグレーション）。
- **共有**：Web Share API / Twitter Web Intent / Clipboard API の3段フォールバック。
- **画像生成**：Canvas API（外部ライブラリ不使用）。
- **演出**：CSSアニメ＋軽量Canvas紙吹雪、効果音は短いdata URI（ON/OFF可）。相棒は絵文字＋CSS or 軽量SVGの表情差分。
- **通知**：起床リマインドにNotification API（未許可時は画面内バナー）。**Web版はアプリ非起動時の通知が弱い**点は割り切り、PWA通知で可能な範囲を狙う。
- **PWA**：ホーム画面追加対応。

### 10.1 コンポーネント構成（案）
```
App
├─ useGameState (localStorage 永続化/マイグレーション)
├─ Setup              … 初回設定＋相棒紹介
├─ TabBar             … ホーム/道/実績/設定
├─ HomeScreen
│   ├─ Mascot         … 相棒の表情＋セリフ
│   ├─ StreakFlame    … 連続記録🔥
│   ├─ WakeButton     … 「起きた！」＋演出
│   ├─ DailyQuest     … コア＋任意ミッションのチェック
│   └─ ScoreResult    … スコア/XP→共有導線
├─ ShareCardModal     … Canvasカード生成＋共有(Web Share/Intent/Copy)
├─ PathScreen         … 早起きの道（進行マップ）
├─ AchievementsScreen … レベル/称号/バッジ/カレンダー/過去カード
└─ SettingsScreen     … 目標/通知/音/共有文/JSON入出力/免責
```

### 10.2 ファイル構成（リポジトリ）
```
jikan-app/                 ← 既存（つもりキャッシュ）は変更しない
docs/wakeup-app/SPEC.md    ← 本設計図
mezamy/
  index.html              ← 実装フェーズで作成（MVP）
  README.md               ← アプリ説明
```

### 10.3 リーグを後で足すなら（フェーズ4の技術メモ）
- 最小構成：ユーザーが任意で表示名＋週間XPだけを送る軽量API（例：サーバレス関数＋KV/DB）。
- 認証は匿名IDで開始 → 必要ならログイン追加。プライバシー上、送信は**オプトイン**。
- ローカル設計を崩さないよう、リーグは「同期レイヤー」を被せる形にする（オフラインでも単体で動く）。

## 12. プライバシー・免責
- データは端末の `localStorage` のみ。共有は**ユーザーが押した時だけ**Twitterへ（自動送信なし）。
- リーグ等でサーバ送信する場合は明確なオプトインとし、送る情報を最小化する。
- 医療機器ではなく診断・治療目的ではない。起床困難が続く等の場合は受診を促す。
